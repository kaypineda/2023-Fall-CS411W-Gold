Risks

Customer Risks

Risk Description

  • C1: User incorrectly inputs data
    • Probability: Low
    • Impact: Medium
  • C2: User doesn't know how to use system
    • Probability: Low
    • Impact: High

Mitigation

  • C1: Edit data later, error notifications
  • C2: Add tutorial reference

Technical Risks

Risk Description

  • T1: Loss of data
    • Probability: Medium
    • Impact: Very High
  • T2: Accidental deletion
    • Probability: Very High
    • Impact: High
  • T3: Loss of internet connection
    • Probability: Medium
    • Impact: Very High

Mitigation

  • T1: Perform regular backups of the database
  • T2: Allow for undo option
  • T3: Allow offline editing for user

Security Risks

Risk Description

  • S1: Data Compromise
    • Probability: Low
    • Impact: High
  • S2: User forgets password
    • Probability: Medium
    • Impact: High
  • S3: Malicious data input
    • Probability: Medium
    • Impact: High

Mitigation

  • S1: Follow industry best practices
  • S2: Allow for password reset
  • S3: Data sanitization
